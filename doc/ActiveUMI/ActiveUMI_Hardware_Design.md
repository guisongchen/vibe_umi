# ActiveUMI 硬件设计方案详解

## 整体架构概览

ActiveUMI硬件系统由三个核心组件组成：
1. **VR夹爪控制器**（VR Gripper Controller）
2. **头戴显示器**（Head-Mounted Display, HMD）
3. **可穿戴计算单元**（Wearable Device）

整体架构如图2所示：
```
┌─────────────────────────────────────────────────────────────┐
│                    ActiveUMI Hardware                       │
├─────────────────┬─────────────────┬─────────────────────────┤
│  VR Controller  │   VR Headset    │      Wearable PC        │
│  (Modified)     │   (Meta Quest3) │      (Backpack)         │
├─────────────────┼─────────────────┼─────────────────────────┤
│ • Gripper       │ • Shoulder      │ • Battery               │
│ • Motor         │   Straps        │ • PC                    │
│ • Fisheye       │ • IR Tracking   │ • Self-contained        │
│   Camera        │ • SLAM System   │   Operation             │
└─────────────────┴─────────────────┴─────────────────────────┘
```

## 1. VR夹爪控制器（VR Gripper Controller）

### 1.1 基础平台
- **设备**: 商业级Meta Quest 3s控制器改装版
- **核心能力**: 同步、低延迟、高精度六自由度（6-DoF）位姿跟踪

### 1.2 位姿跟踪原理
```
跟踪流程：
┌──────────────────────────────────────────────────────┐
│  头戴式耳机板载相机                                  │
│       ↓                                              │
│  识别控制器上的红外LED独特图案                        │
│       ↓                                              │
│  实时三角测量计算控制器位姿                          │
│       ↓                                              │
│  获取6-DoF数据 (x, y, z, roll, pitch, yaw)          │
│       ↓                                              │
│  刚性安装到目标机器人 → 控制器位姿=机器人末端位姿     │
└──────────────────────────────────────────────────────┘
```

### 1.3 夹爪执行器（Gripper Actuation）
| 特性 | 设计细节 |
|-----|---------|
| 驱动方式 | 微电机直接集成到控制器 |
| 控制方式 | 操作者直观控制机器人夹爪开合 |
| 关键优势 | **非侵入式**：不替换机器人原有夹爪，而是在控制器上附加相同副本 |
| 适配性 | 可部署于多种现成机器人，最小化改装需求 |

### 1.4 鱼眼相机（Fisheye Camera）
| 参数 | 规格 |
|-----|------|
| 安装位置 | 每个控制器上 |
| 视角 | 最大化视野范围 |
| 功能 | 捕获机器人操作环境的丰富视觉信息 |
| 作用 | 作为头戴相机第一人称视角的补充，提供"手腕视角"（Wrist View） |

## 2. 头戴显示器（Head-Mounted Display）

### 2.1 设备选型
- **设备**: Meta Quest 3s HMD
- **佩戴方式**: 肩带式固定（Shoulder Straps）

### 2.2 双重关键作用

#### 作用一：高精度定位中心（Localization Hub）
| 功能 | 技术细节 |
|-----|---------|
| SLAM系统 | 提供稳定可靠的世界坐标系 |
| 跟踪能力 | 同时跟踪操作者头部和控制器6-DoF位姿 |
| 前置相机 | 作为动态顶部相机，提供全局视角 |

#### 作用二：沉浸式数据收集（Immersive Data Collection）
```
VR环境渲染：
┌────────────────────────────────────────────────────┐
│                  VR Environment                    │
│  ┌────────────────────────────────────────────┐   │
│  │         3D模型：机器人双臂                 │   │
│  │              ↕ 精确对齐                     │   │
│  │     操作者手持控制器（对应机器人夹爪）       │   │
│  │              ↓                              │   │
│  │    实时可视化机器人运动                     │   │
│  │    操作者可清晰观察机器人动作               │   │
│  └────────────────────────────────────────────┘   │
└────────────────────────────────────────────────────┘
```

### 2.3 主动感知输入
- 实时记录操作者头戴显示器（HMD）的6-DoF位姿
- 作为策略的额外输入
- 学习操作者头部运动（视觉注意力）与手部动作的关联

## 3. 可穿戴计算单元（Wearable Device）

### 3.1 设计目标
- 支持在任何环境中进行数据收集
- 解放操作者，摆脱固定工作站
- 允许自由移动，跨多样化场景收集数据

### 3.2 硬件组成
| 组件 | 规格/功能 |
|-----|----------|
| 小型计算机 | 背负式佩戴 |
| 电池 | 独立供电 |
| 背包式设计 | 自包含、便携 |

### 3.3 数据流架构
```
┌─────────────────────────────────────────────────────────────┐
│                    Data Collection                         │
│                       30Hz                                 │
├─────────────────────────────────────────────────────────────┤
│  Datasets:                                                  │
│  ├── Language: Task descriptions                           │
│  ├── Action:                                               │
│  │   ├── Joint: 6 DoF × 2 arms                             │
│  │   └── Gripper: 1 DoF × 2 grippers                      │
│  └── Camera:                                               │
│      ├── Wrist camera × 2 (Left/Right)  ← 附加鱼眼相机     │
│      └── Top camera × 1 (Head-mounted)  ← VR自带前置相机   │
└─────────────────────────────────────────────────────────────┘
```

#### 相机系统详细说明

| 相机 | 来源 | 安装方式 | 功能 |
|-----|------|---------|------|
| **手腕相机** × 2 | **附加安装** | 鱼眼相机固定在左右控制器上 | 提供"手腕视角"，捕获操作环境丰富视觉信息 |
| **顶部相机** × 1 | **VR自带** | Meta Quest 3s 前置彩色相机 | 作为动态顶部相机，提供全局视角，与操作者视线耦合 |

> **重要说明**：顶部相机是VR头显自带的前置相机，**无需额外安装或标定**。这是ActiveUMI相比其他需要额外安装顶部相机的系统的优势之一。

## 4. 校准系统（Calibration System）

为确保数据质量和精度，ActiveUMI实现了**三种关键校准方法**，形成完整的校准流程：

```
┌─────────────────────────────────────────────────────────┐
│                    校准流程（三级递进）                  │
├─────────────────────────────────────────────────────────┤
│  Step 1: 现场环境设置（粗对准）                          │
│     ↓ 按下B按钮，对齐虚拟/物理坐标系                     │
│                                                        │
│  Step 2: 夹爪定位器（精确定位）                          │
│     ↓ 控制器放入物理夹具，自动校准初始姿态               │
│                                                        │
│  Step 3: 零位触觉反馈（精调辅助）                        │
│     ↓ 振动提示辅助精确定位                              │
│                                                        │
│  结果：精确一致的初始姿态，准备开始数据收集              │
└─────────────────────────────────────────────────────────┘
```

### 4.1 现场环境设置（In-Situ Environment Setup）

#### 目的
- 重置6-DoF零点，重新定位基础坐标系
- 建立**虚拟世界坐标系**与**实际物理工作空间**的对应关系
- 允许在任何新环境中快速开始数据收集

#### 操作流程
```
操作者按下控制器 "B" 按钮
        ↓
触发基础坐标系重新定位
        ↓
VR头显中实时渲染坐标轴（x, y, z三个方向）
        ↓
操作者通过头显观察虚拟坐标系
        ↓
直观对齐虚拟参考系与物理工作空间
        ↓
确认对齐 → 建立统一的世界坐标系
```

#### 技术细节
| 特性 | 说明 |
|-----|------|
| 触发方式 | 按下控制器"B"按钮 |
| 视觉辅助 | 坐标轴实时渲染在头戴显示器中 |
| 对齐方式 | 操作者直观观察并调整 |
| 精度等级 | **粗对准** —— 为后续精确定位提供基础 |
| 优势 | 可在任何环境中灵活开始数据收集，不受限于固定实验室环境 |

### 4.2 夹爪定位器（Gripper Placeholder）

#### 硬件设计
```
┌────────────────────────────────────────────────────┐
│              Gripper Placeholder                   │
│                  (物理夹具/停靠站)                  │
├────────────────────────────────────────────────────┤
│  结构特点：                                         │
│  • 可放置在工作区任意位置                            │
│  • 控制器放入时，相对距离和姿态固定为预定义状态       │
│  • 配备一个指定按钮用于触发校准                      │
└────────────────────────────────────────────────────┘
```

#### 核心原理
> *"When the controllers are seated in the placeholder, their relative distance and pose are **fixed to a predefined state**. Pressing a designated button while the controllers are docked **instantly calibrates the virtual coordinate system**, aligning its origin and orientation with **this known physical configuration**.*"

**关键点**：
- **预定义状态**：夹具机械结构保证每次放入时，控制器的位姿是已知且固定的
- **已知物理配置**：这个预定义的位姿在虚拟世界中有对应的坐标
- **一键校准**：按下按钮后，系统自动将虚拟坐标系与这个已知的物理配置对齐

#### 校准流程
```
┌─────────────────────────────────────────────────────────┐
│              夹爪定位器校准步骤                          │
├─────────────────────────────────────────────────────────┤
│  1. 放置                                               │
│     操作者将VR控制器放入物理夹具                        │
│              ↓                                         │
│     控制器相对距离和姿态 → 固定为预定义状态              │
│                                                        │
│  2. 触发                                               │
│     按下夹具上的指定按钮                                │
│              ↓                                         │
│     瞬间完成虚拟坐标系校准                              │
│              ↓                                         │
│     虚拟坐标系的原点和方向 ←→ 与已知物理配置对齐         │
│                                                        │
│  3. 结果                                               │
│     确保每次数据收集从精确一致的初始姿态开始            │
└─────────────────────────────────────────────────────────┘
```

#### 标定效果
| 校准前 | 校准后 |
|-------|-------|
| 虚拟世界坐标系（任意初始状态） | 与物理世界精确对齐 |
| 坐标原点可能偏移 | 原点 = 预定义位姿 |
| 方向可能不一致 | 方向 = 已知物理配置 |

### 4.3 零位触觉反馈（Haptic Feedback for Zero-Point）

#### 工作原理
| 参数 | 设置 |
|-----|------|
| 触发条件 | 夹爪接近零位（原点）**3cm范围内** |
| 反馈方式 | 控制器微电机产生**高频振动** |
| 目的 | 提醒操作者夹爪接近基础位置 |

#### 操作流程
```
操作者移动夹爪
        ↓
进入零位3cm范围内
        ↓
微电机产生高频振动（触觉提示）
        ↓
操作者感知振动，确认接近基准位置
        ↓
无需查看数值读数，完成精确定位
```

#### 设计优势
| 优势 | 说明 |
|-----|------|
| 盲操作支持 | 无需查看数值读数，可"盲"对齐 |
| 效率提升 | 显著提升校准速度和效率 |
| 用户体验 | 减少操作者的认知负担 |
| 精度辅助 | 作为前两种校准方法的精调补充 |

---

### 4.4 三种校准方法对比

| 校准方法 | 作用对象 | 操作方式 | 精度等级 | 主要目的 |
|---------|---------|---------|---------|---------|
| **现场环境设置** | 世界坐标系原点 | 按B按钮 + 视觉对齐虚拟/物理坐标轴 | 粗对准 | 建立虚拟与物理的初步对应 |
| **夹爪定位器** | 控制器初始位姿 | 放入物理夹具 + 按指定按钮 | 精确定位 | 确保每次从一致的初始姿态开始 |
| **零位触觉反馈** | 辅助精确定位 | 接近零位时振动提示 | 精调辅助 | 提升校准效率和用户体验 |

---

### 4.5 校准精度对比

| 系统 | 相对位姿误差（RPE） |
|-----|-------------------|
| UMI | 10.1 mm |
| **ActiveUMI** | **4.0 mm** |

> **分析**：VR系统的内置SLAM跟踪和高精度6-DoF位姿估计带来更低的误差，从而获得更好的数据质量和策略训练效果。ActiveUMI的RPE比UMI低约**60%**。

### 4.6 系统集成标定说明

#### 需要标定的内容
| 标定项目 | 论文提及 | 标定方式 |
|---------|---------|---------|
| 控制器-世界坐标系 | ✅ | 现场环境设置（B按钮）+ 夹爪定位器 |
| 夹爪初始位姿 | ✅ | 夹爪定位器（物理夹具）|
| 头显-控制器相对位姿 | ✅ | VR系统自带SLAM自动跟踪 |
| 鱼眼相机-控制器外参 | ❌未明确 | 可能通过机械固定/预标定 |

#### 相机系统说明
- **顶部相机**：VR头显（Meta Quest 3s）**自带的前置彩色相机**，无需额外标定
- **手腕相机**：附加的鱼眼相机，可能通过**机械结构设计**保证与控制器的固定几何关系

## 5. 硬件设计优势总结

| 优势 | 说明 |
|-----|------|
| **硬件灵活性** | 相比固定夹爪的系统，通过改装Meta Quest控制器适配不同机器人 |
| **非侵入式** | 不替换机器人原有夹爪，仅附加相同副本 |
| **便携性** | 背包式自包含设计，可在野外自由移动 |
| **高精度** | 4mm位姿误差，优于传统UMI系统 |
| **低成本** | 基于消费级VR设备（Meta Quest 3s） |
| **主动感知** | 头戴显示器记录头部运动，实现视点控制 |
