## 2. 主动感知策略学习

### 2.1 核心概念

传统 UMI 系统的局限性：
- 依赖手腕安装相机
- 相机视角受操作需求限制
- 难以处理遮挡、长时程任务

ActiveUMI 解决方案：
- **记录操作者的头部移动**（HMD 6-DoF位姿）
- **学习视觉注意力与手部动作的关联**
- **部署时机器人主动控制视点**

### 2.2 策略学习架构

```
┌─────────────────────────────────────────────────────────────────┐
│              Active Perception Policy Learning                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Training Phase:                                                │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  Inputs:                                                 │   │
│  │  • Head 6-DoF pose (HMD)  ← 操作者头部运动               │   │
│  │  • Hand 6-DoF poses (L/R) ← 操作者手部动作               │   │
│  │  • Wrist camera images (L/R)                            │   │
│  │  • Top camera image (HMD)                               │   │
│  │  • Language instruction                                 │   │
│  │                                                          │   │
│  │  Learning:                                               │   │
│  │  Head movement (attention) ⟷ Hand action correlation    │   │
│  └─────────────────────────────────────────────────────────┘   │
│                          ↓                                      │
│  Policy Network (π₀ VLA Model)                                  │
│                          ↓                                      │
│  Deployment Phase:                                              │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  Policy predicts:                                      │   │
│  │  • Robot head 6-DoF pose ← 主动视点控制                 │   │
│  │  • Robot arm actions                                    │   │
│  │                                                          │   │
│  │  Execution:                                              │   │
│  │  • Low-level controller drives robot head movement      │   │
│  │  • Dynamically adjust viewpoint                         │   │
│  │  • Overcome occlusions                                  │   │
│  │  • Acquire task-critical information on demand          │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 策略模型配置

| 参数 | 配置 |
|------|------|
| **基础模型** | π₀ (Vision-Language-Action Model) |
| **输入模态** | Vision + Language + Action History |
| **输出** | 6-DoF Head Pose + 6-DoF×2 Arm Actions + 1-DoF×2 Grippers |
| **微调迭代** | 50k iterations |
| **学习率调度** | Cosine learning rate scheduler |

## 4. 混合训练策略

### 4.1 核心发现

ActiveUMI数据具有高样本效率，可与少量遥操作数据混合训练：

| 遥操作数据比例 | ActiveUMI数据比例 | 平均成功率 |
|--------------|-----------------|----------|
| 0% | 100% | 80% |
| **1%** | **99%** | **95%** ⭐ |
| 10% | 90% | 90% |

**关键洞察**：仅添加 **1%** 的遥操作数据即可将成功率从80%提升至95%，表明ActiveUMI数据质量高，少量真实机器人数据即可有效微调策略。

│  │  • 成本低，数据收集效率高                              │   │

    ↓

## 7. 性能指标与评估

### 7.1 数据收集效率对比

| 方法 | 绳子装箱时间 | 叠衬衫时间 | 相对效率 |
|------|------------|-----------|---------|
| **人类徒手** | 基准 | 基准 | 1.0× (最快) |
| **ActiveUMI** | 慢 **2.06×** | 慢 **1.49×** | ~0.5-0.7× |
| **传统遥操作** | 慢 **3.27×** | 慢 **2.63×** | ~0.3-0.4× |

**结论**：ActiveUMI在保持数据质量的同时，显著快于传统遥操作。

### 7.2 数据精度对比

| 指标 | UMI | ActiveUMI | 改进 |
|------|-----|-----------|------|
| **相对位姿误差 (RPE)** | 10.1 mm | **4.0 mm** | ↓ 60% |

### 7.3 数据精度来源分析

**ActiveUMI的数据精度主要依赖VR组件内置的定位算法**。

论文在第4.5节明确指出：

> *"This low error is **naturally comes from the advantage of the VR system**, thus we obtain much better data quality"*

（翻译：这种低误差**自然来自于VR系统的优势**，因此我们获得了更好的数据质量）

#### VR系统提供的核心定位能力

| 组件 | 定位技术 | 精度贡献 |
|------|---------|---------|
| **Meta Quest 3s 头显** | Inside-out Tracking | 头显内置相机跟踪环境特征，提供稳定的world坐标系 |
| **控制器跟踪** | IR LED + 视觉三角测量 | 头显相机识别控制器上独特的红外LED图案，实时计算6-DoF位姿 |
| **SLAM系统** | 同步定位与地图构建 | 提供稳定可靠的世界坐标系参考 |

#### VR内置定位流程

```
┌─────────────────────────────────────────────────────────┐
│              VR内置定位系统 (Meta Quest 3s)              │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  头显相机 (Onboard Cameras)                             │
│       ↓                                                 │
│  识别控制器IR LED独特图案 (Unique IR LED Pattern)       │
│       ↓                                                 │
│  实时三角测量 (Real-time Triangulation)                 │
│       ↓                                                 │
│  输出6-DoF位姿数据 (x, y, z, roll, pitch, yaw)         │
│       ↓                                                 │
│  SLAM系统优化与坐标系统一                               │
│                                                         │
└─────────────────────────────────────────────────────────┘
         ↓
    ActiveUMI记录数据
         ↓
    RPE: 4.0mm (vs UMI: 10.1mm)
```

#### ActiveUMI附加的精度保障

虽然基础精度来自VR系统，但ActiveUMI通过**校准系统**确保数据**一致性**和**准确性**：

| 校准方法 | 作用 | 与VR定位的关系 |
|---------|------|---------------|
| **现场环境设置** | 重置坐标系原点 | 利用VR的SLAM系统建立world坐标系 |
| **夹爪定位器** | 确保初始姿态一致 | 在VR跟踪基础上提供物理参考 |
| **零位触觉反馈** | 精调辅助 | 辅助操作者利用VR的高精度定位 |

#### 对比：UMI为什么精度较低？

| 系统 | 定位方式 | 误差 |
|------|---------|------|
| **UMI** | 外部相机 + 手眼标定 | 10.1mm |
| **ActiveUMI** | VR内置Inside-out Tracking | **4.0mm** |

**关键差异**：
- UMI依赖外部相机和可能的手眼标定误差
- ActiveUMI直接使用VR系统成熟的、经过优化的定位算法

#### 结论

ActiveUMI的**4.0mm RPE精度**主要归功于：
1. **Meta Quest 3s成熟的Inside-out Tracking技术**
2. **低延迟、高精度的6-DoF位姿估计**
3. **集成的SLAM系统提供的稳定坐标系**

ActiveUMI的贡献在于**利用**这一成熟的高精度定位系统，并通过校准流程确保数据收集的一致性和可用性，而不是重新实现定位算法。
