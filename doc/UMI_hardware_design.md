# UMI 硬件设计详解

## 概述

UMI (Universal Manipulation Interface) 是一个**触发式手持3D打印平行夹爪**，配有软手指，安装有 **GoPro相机作为唯一的传感器和记录设备**。对于双手操作，可以简单地添加另一个夹爪进行扩展。

> **核心设计问题**：如何仅用单个手腕安装的相机，为多种任务捕获足够的信息？

## 六大硬件设计要点 (HD1-HD6)

### HD1. 手腕安装相机作为输入观察

**核心设计**：完全依赖手腕安装的相机，无需任何外部相机设置。

当在机器人上部署UMI时，GoPro相机放置在与手持夹爪相同的位置（相对于3D打印手指）。

**四大优势**：

| 优势 | 说明 |
|------|------|
| **最小化观察形态差异** | 人类演示和机器人部署时相机位置几乎相同，使策略输入对形态差异不敏感 |
| **机械鲁棒性** | 相机与手指机械固定，机器人上安装UMI时无需相机-机器人-世界标定，系统对机械冲击更鲁棒 |
| **便携硬件设置** | 无需外部静态相机或额外板载计算，简化数据收集，整个系统高度便携 |
| **相机运动实现自然数据多样化** | 移动相机训练时，策略学会关注任务相关物体而非背景结构，增强对干扰物的鲁棒性 |

---

### HD2. 鱼眼镜头提供视觉上下文

- **155°超广角鱼眼镜头**安装在手腕式GoPro相机上
- **直接使用原始鱼眼图像**，不进行去畸变处理
- 鱼眼效果在**中心保持分辨率**，同时压缩边缘信息
- 相比针孔模型，避免了宽FoV导致的极端畸变

**关键特性**：
- 原始鱼眼图像保留中心分辨率，边缘信息被压缩
- 反畸变后的针孔图像（图3右侧）会产生极端畸变，不适合学习

---

### HD3. 侧面镜子实现隐式立体视觉

这是UMI**最具创新性的设计**之一：

```
                    超广角155° FoV
                         │
    ┌────────────────────┼────────────────────┐
    │       左镜         │         右镜       │
    │    (虚拟相机)      │  主相机 │  (虚拟相机)  │
    │                    │         │              │
    └────────────────────┴─────────┴──────────────┘
```

**工作原理**：

1. 在相机外围视野放置一对**物理镜子**
2. 镜子中的图像相当于从镜子平面反射的**额外虚拟相机**看到的画面
3. 创建**隐式立体视图**，所有内容都在同一图像中
4. 需要对镜子中的内容进行**数字反射**处理

**数字反射的重要性**：

- 没有数字反射时，镜子中观察到的物体朝向与主相机相反
- 经过数字反射后，物体朝向在所有三个视图中保持一致
- 例如：杯子把手在主视图和两侧镜子中朝向一致

**效果**：
- 直接提供镜子图像会降低性能（18/20=90% → 17/20=85%）
- 数字反射后性能达到 **20/20=100%**

### HD4. IMU感知跟踪

- 利用GoPro内置的**IMU数据**（加速度计和陀螺仪）
- 记录为标准mp4视频文件
- 结合**视觉-惯性ORB-SLAM3**系统

**关键优势**：
- 即使视觉跟踪失败（如运动模糊或缺乏视觉特征，例如看向桌面时），也能在短时间内保持跟踪
- 联合视觉-惯性优化允许直接恢复真实公制尺度，对动作精度和夹爪姿态感知至关重要

---

### HD5. 连续夹爪控制

与先前工作的**二进制开/合动作**不同：

**设计特点**：
- **连续跟踪手指宽度**，通过基准标记（fiducial markers）实现（图2左侧）
- 使用**串联弹性末端执行器原理**
- 通过调节软手指变形来**隐式记录和控制抓取力**

**显著扩展任务范围**：
- **抛掷任务**：需要精确释放时机，二进制夹爪动作无法满足精度要求
- 不同宽度的物体需要不同的抓取策略
- 通过连续控制夹爪宽度，可以调节软手指变形，从而控制抓取力

---

### HD6. 基于运动学的数据过滤

**设计原则**：数据收集过程是**机器人无关的**

**处理流程**：
1. 针对特定机器人形态应用**运动学数据过滤**
2. 当机器人基座位置和运动学已知时，利用SLAM恢复的绝对末端执行器姿态
3. 进行**运动学和动力学可行性过滤**
4. 在过滤后的数据集上训练，确保策略符合特定机器人的运动学约束

## 双手扩展

UMI可轻松扩展为双手设置：

- 添加第二个UMI夹爪
- 两个夹爪通过**共享坐标系**计算相对姿态
- 实现双手协调任务（如叠衣服）

**关键**：提供两个夹爪之间的相对姿态对于双手协调和任务成功至关重要。

---

## 软手指设计

**材料**：95A TPU材料3D打印

**结构特点**：
- 肋条状图案维持指尖刚性
- 同时符合物体几何形状，实现更安全的抓取
- 提供**被动机械顺应性**

**应用场景**：
- 接触丰富任务（如打开水龙头、从桌面 scoop 薄衣服）
- 软手指在结合连续夹爪控制时，提供隐式抓取力控制
- 可安全抓取生鸡蛋而不破裂（图A3）

## 机器人部署适配

### Franka机器人适配

由于Franka FR2的末端执行器俯仰角有限（FR2设计用于自上而下pick-and-place，而UMI夹爪主要水平手持）：

- 设计并3D打印了**自定义安装适配器**
- 将WSG50夹爪相对于机器人末端执行器法兰旋转**90度**

### 通用适配

UMI框架可应用于：
- 移动操作器（mobile manipulators）
- 不同自由度机器人（6DoF、7DoF机械臂）

## 关键创新总结

| 创新点 | 说明 |
|--------|------|
| **单传感器设计** | 仅用GoPro实现视觉+IMU+记录，简化硬件 |
| **镜子实现立体视觉** | 无需额外相机即可获得深度感知 |
| **手腕安装视角** | 最小化人与机器人的观察差异 |
| **低成本开源** | 硬件成本低于$400，完全开源 |
| **连续夹爪控制** | 扩展任务范围到动态、精确操作 |
